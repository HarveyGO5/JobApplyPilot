# Publishing Checklist for GitHub Product Promotion

## 1. Repository setup

- Add `README.md` with product value, features, and direct CTA links
- Upload and reference clear visual assets from `assets/`
- Add `index.html` for GitHub Pages landing page

## 2. Links and conversion paths

- Chrome Web Store link:
  `https://chromewebstore.google.com/detail/jobapplypilot-linkedin-au/imlockpnecjpgiijhacadkamfhmedmac`
- Official website link:
  `https://apply-pilot-web.applypilot.workers.dev/`
- Ensure both links appear:
  - at top of README
  - in hero CTA buttons
  - near the bottom of the page

## 3. Search and LLM visibility

- Keep one clear canonical product sentence across all channels
- Include short-tail and long-tail keywords naturally
- Add JSON-LD SoftwareApplication schema in `index.html`
- Include platform-specific terms (LinkedIn, Workday, Greenhouse, iCIMS)

## 4. Visual quality

- Use one hero image and one workflow image minimum
- Keep image filenames in lowercase English with hyphens
- Optimize image size if page load becomes slow

## 5. Trust and clarity

- Avoid exaggerated claims
- Include a responsible-use section
- Keep feature descriptions aligned with real product behavior

## 6. Post-publish distribution

- Share GitHub URL in product communities and job seeker communities
- Use one message template per channel with keyword variation
- Track weekly changes in:
  - GitHub stars
  - referral visits
  - Chrome store clicks

## 7. Iteration cadence

- Week 1: ship page and collect baseline metrics
- Week 2: update headline and CTA copy
- Week 3: add one new real-world use case section
- Week 4: refresh screenshots and expand FAQ
